# 문서 작성 가이드

- 한글로 작성해야 합니다.
- 문서는 반드시 200줄 이하로 완성합니다.

# 코드 작성 가이드

- 함수를 작성할 때, 함수 내부에 논리 단위로 주석을 작성해주세요.
- 주석은 핵심만 담아서 짧고 간결하게 한 문장으로 써야합니다.

# 서비스 코드 작성 가이드

- application/services 코드를 작성할 때 다음의 세부 사항을 고려하세요.

- TDD 원칙에 따라서 개발하려고 하는 기능의 시나리오(기획 세부 명세)를 해피패스만을 고려해서 테스트 코드를 먼저 작성하고, 그 테스트를 실패시킨 후, 해피패스를 통과하기 위한 서비스 코드를 작성해주세요.

- 위 작업이 완료되면 개발자에게 검토를 요청하세요.

- 해피패스를 구현해서 개발자와 합의가 끝나면, 다음 테스트 코드를 작성할 때, 가장 크리티컬한 문제가 있는 케이스에 대해서만 최대 2개의 테스트 케이스 시나리오를 먼저 제안하고 개발자와 어떤 케이스를 테스트 케이스로 작성할지 의논해주세요.

- 개발자와 합의가 되는 경우에 한해서 실제로 테스트 케이스를 작성하고, 서비스 코드가 테스트를 통과할 수 있도록 수정합니다.

- 작업이 종료되면 반드시 npm run type 커맨드를 실행해서 문제를 해결해주세요.

# 추가 주의사항

- 의존성을 주입받는 모든 객체 또는 함수는, 반드시 계약서로 인터페이스가 존재해야 합니다. 혹시 구현이 안되어 있다면 작업을 중단하고 알려주세요. 문제가 없다면 서비스 코드 작성을 완료하세요.

- 로직을 구현한 후에는 npm run test 커맨드를 사용해서 테스트를 검증해주세요.

- 테스트 통과에 실패하는 경우 테스트 실패 원인이 새롭게 작성한 테스트 코드에서 실패했다면 직접 다시 수정하세요. 단, 기존의 테스트 코드에서 실패했다면, 그 경우에는 작업을 중단하고 원인을 분석해서 간단하게 알려주세요.
